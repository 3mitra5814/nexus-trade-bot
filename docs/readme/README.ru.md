## Сначала прочитайте это

Если вы обычный пользователь, начните с установки одной командой, откройте веб-консоль, добавьте API-ключ только с правом торговли и без вывода средств, затем запустите небольшого тестового бота.

Если вы разработчик, начните со сборки из исходников, посмотрите `config.example.yaml`, выполните `go test ./...`, а затем используйте worker mode для запуска бота с конкретным конфигом.

README идет в практическом порядке: установка, биржи, возможности, примеры стратегий, параметры, ручная установка и проверки перед реальной торговлей.

## Установка одной командой

Запустите это на новом сервере Ubuntu:

```bash
wget -O nexus-trade-bot.sh https://raw.githubusercontent.com/haohaoi34/nexus-trade-bot/main/scripts/nexus-trade-bot.sh && chmod +x nexus-trade-bot.sh && ./nexus-trade-bot.sh install && ./nexus-trade-bot.sh start
```

Сервер-раннер автоматически:

- Устанавливает недостающие зависимости Ubuntu.
- Устанавливает Go, если на сервере нет совместимой версии.
- Клонирует `https://github.com/haohaoi34/nexus-trade-bot.git`, если он еще не запущен внутри проверки исходного кода.
- Собирает бота из исходного кода или использует прилагаемый двоичный файл в пакете выпуска.
- При необходимости создает `config.yaml` из `config.example.yaml` и сохраняет его локально.
- Запускает веб-консоль в фоновом режиме и записывает логи в `logs/`.
- Выводит понятный блок доступа с локальным URL, адресом привязки, PID-файлом и путями к журналам; удаленный доступ отключен, пока вы явно не зададите публичную привязку.

Полезные команды сервера:

```bash
./nexus-trade-bot.sh install
./nexus-trade-bot.sh start
./nexus-trade-bot.sh status
./nexus-trade-bot.sh logs
./nexus-trade-bot.sh restart
./nexus-trade-bot.sh stop
./nexus-trade-bot.sh update
```

Веб-логин по умолчанию:

```text
username: admin
password: admin
```

Измените пароль по умолчанию сразу после первого входа в систему.

## Что он делает

nexus-trade-bot поможет вам запускать сеточные стратегии из чистой веб-консоли:

- Добавьте API-интерфейсы обмена один раз и проверьте их перед использованием.
- Создавайте несколько ботов для разных символов, аккаунтов и направлений.
- Выбирайте фьючерс или спот. Фьючерс выбран по умолчанию.
- Используйте длинный, короткий или нейтральный режим для фьючерсов; используйте длинный режим на месте.
- Автоматически загружайте спотовые символы Binance, Bitget, Bybit, OKX, Gate и Hyperliquid.
- Следите за балансом, объемом торгов, статусом бота и PnL в режиме реального времени.
- Приостановите работу бота, измените параметры и перезапустите его с последними настройками.
- Позвольте монитору рисков остановить торговлю во время аномальных движений рынка.

Он предназначен для трейдеров, которые заботятся об исполнении, обороте и контроле, а не для людей, которые хотят редактировать файлы конфигурации весь день.

## Основная идея

Сетчатый бот размещает ордера на покупку и продажу через фиксированные интервалы цен. Вместо того, чтобы пытаться предсказать точную вершину или дно, он продолжает работать в диапазоне цен:

- Когда цена падает, бот постепенно покупает в соответствии с настройками вашей сетки.
- Когда цена восстанавливается, бот шаг за шагом продает более высокие уровни.
- На боковом или восходящем рынке это может превратить волатильность в повторяющиеся реализованные сделки.
- При одностороннем нисходящем тренде бот накапливает позицию и нуждается в достаточной марже, лимитах риска и терпении.

Целью не является волшебная прибыль. Целью является дисциплинированное исполнение: постоянное расстояние между ордерами, контролируемый размер ордера, видимый риск и автоматическая реакция, когда рынок становится ненормальным.


## Пример стратегии: сетка ETH с высоким оборотом

Вот практический пример, позволяющий понять, как трейдеры используют этот тип бота.

Предположим, что ETH торгуется рядом с `3000`, и вы настраиваете:

| Параметр | Пример |
| --- | --- |
| Символ | `ETHUSDT` или `ETHUSDC` |
| Направление | Длинная сетка |
| Ценовой интервал | `1 USDT` |
| Сумма заказа | `300 USDT` для каждого порядка сетки |
| Рыночный стиль | Боковой или восходящий рынок |

Благодаря ограниченному интервалу `1 USDT` и активной ликвидности ETH бот может генерировать очень высокий оборот. На оживленном рынке такая конфигурация может достигать миллионов долларов в ежедневном объеме торгов и десятков миллионов в ежемесячном объеме, в зависимости от волатильности, комиссий, ликвидности и размера счета.

Вот почему многие трейдеры используют сеточные системы для двух целей:

- **Увеличение объема**: увеличение объема торговли фьючерсами для VIP-уровней или кампаний биржи.
- **Сбор волатильности**: постоянные покупки ниже и продажи выше внутри диапазона.


## Пример логики просадки

Сеточную торговлю необходимо планировать с учетом просадки.

Предположим, что ETH начинается около `3000` и падает до `2700`. Длинная сетка обычно несет плавающий убыток, поскольку на пути вниз она совершала покупки. Но он также накопил более низкие записи. Если позже цена отскочит от `2700` к `2850`, средняя стоимость может снизиться настолько, что счет приблизится к безубыточности раньше, чем при единственной записи на `3000`.

Если ETH вернется близко к исходной области `3000`, стратегия может выиграть от обоих:

- восстановление запасов после отскока;
- реализованы спреды сетки, собранные во время движения.

Некоторые трейдеры резервируют больший буфер маржи, например, вокруг `30,000 USDT`, чтобы создать сетку, которая может выдержать гораздо более глубокое движение, такое как просадка `1000 USDT` ETH. Достаточно ли этого, зависит от кредитного плеча, режима маржи, размера позиции, комиссий, правил поддержания маржи на бирже и того, насколько агрессивна ваша сетка.

Важный момент: прибыль от сети приходит от подготовки, а не от оптимизма. Прежде чем определять размер, подсчитайте, как далеко рынок может двигаться против вас, какую позицию может накопить бот и что произойдет, если рынок не восстановится быстро.


## Встроенная защита от рисков

Быстрые односторонние падения — худшая среда для агрессивной длинной сетки. nexus-trade-bot включает в себя монитор рыночных рисков, предназначенный для решения этой проблемы:

- следит за основными символами, такими как BTC, ETH, SOL, XRP и DOGE;
- обнаруживает аномальное поведение цен и объемов;
- приостанавливает торговлю, когда рыночные условия становятся опасными;
- позволяет торговать снова только после восстановления достаточного количества отслеживаемых символов.

Это не устраняет риск, но дает боту возможность прекратить увеличивать риски во время внезапных движений в стиле ликвидации.


## Распространенные способы его использования

### 1. Создание объемов и VIP-уровней

Используйте узкие интервалы и контролируемый размер ордера для инструментов с высокой ликвидностью. Целью является высокая текучесть кадров и предсказуемое исполнение. Ставки комиссий здесь имеют большое значение, поэтому по возможности используйте пары с низкой комиссией или программы скидок.

### 2. Длинная сетка после отката рынка

Начинайте после значительного падения, а не гонитесь за вертикальным насосом. Бот покупает слоями и продает на подборах. Этот стиль требует достаточного запаса, чтобы пережить более глубокие откаты.

### 3. Сетка Binance Spot

Используйте спотовый режим, если вы хотите, чтобы бот покупал и продавал реальные монеты вместо открытия фьючерсных позиций с кредитным плечом. Спотовый режим предназначен только для длинных позиций: бот сначала покупает более низкие уровни, а затем продает инвентарь на подбор. Это проще, чем фьючерсы, но все же требуется достаточный баланс котировок и план на случай продолжительного нисходящего тренда.

### 4. Выход из инвентаря

Если у вас уже есть позиция, бот может помочь постепенно продать ее по мере роста цены. Когда позиция полностью сократится, можно остановить бота.

### 5. Нейтральная сетка

Используйте нейтральный режим, если вам нужно поведение сетки как на длинной, так и на короткой стороне. Начните с меньшего размера и посмотрите, как биржа обрабатывает режим позиции перед масштабированием.


### 6. Классическая сетка

Классическая сетка работает только на фьючерсах в нейтральном режиме. Она держит 50 активных ордеров на покупку ниже текущей сеточной цены и 50 активных ордеров на продажу выше нее, без верхней и нижней границы. Цель — 100 активных ордеров всего; исполненные уровни автоматически пополняются. Hyperliquid futures пока не поддерживается, потому что для режима нужна нейтральная двусторонняя позиция.

## Руководство по параметрам

| Настройка | Что это значит | Практический совет |
| --- | --- | --- |
| `symbol` | Торговая пара | Начните с ликвидных пар, таких как BTC или ETH. |
| `app.market_type` | `futures` или `spot` | По умолчанию `futures`. Спотовая торговля в режиме реального времени поддерживает Binance, Bitget, Bybit, OKX, Gate и Hyperliquid через специальные адаптеры. |
| `mode` | `normal`, `aggressive` или `classic` | `classic` принудительно включает futures + neutral и держит цель 100 активных ордеров: 50 покупок и 50 продаж. |
| `direction` | `long`, `short` или `neutral` | Длинным сеткам нужен запас на просадки. Короткие сетки не должны случайно принимать несвязанную короткую позицию вручную, если вы намеренно не включите такое поведение. |
| `price_interval` | Расстояние между уровнями сетки | Меньший интервал означает больше сделок и больше комиссий. |
| `order_quantity` | Сумма, использованная на заказ | Большая сумма увеличивает оборот и просадку. Убедитесь, что пользовательский интерфейс показывает значение котировки или базовое количество для вашей биржи и типа рынка. |
| `min_order_value` | Условный минимальный заказ | Должен соответствовать биржевым минимумам. |
| `risk_control.enabled` | Защита от рыночных аномалий | Оставьте его включенным, если вы точно не знаете, почему нет. |

Английский, упрощенный китайский, русский, корейский, японский, испанский, вьетнамский, хинди, португальский, арабский и традиционный китайский.

В режиме веб-консоли отображается:

- Управление API

## Ручная установка

```bash
git clone https://github.com/haohaoi34/nexus-trade-bot.git
cd nexus-trade-bot
go mod download
go build -o nexus-trade-bot .
```

Запустите веб-консоль:

```bash
./nexus-trade-bot
```

Локальный URL-адрес по умолчанию:

```text
http://127.0.0.1:8080
```

Выставить на сервере:

```bash
NEXUS_TRADE_BOT_ADDR=0.0.0.0:8080 ./nexus-trade-bot
```

Запуск сервера одной командой из исходной проверки:

```bash
chmod +x scripts/nexus-trade-bot.sh
scripts/nexus-trade-bot.sh install
scripts/nexus-trade-bot.sh start
scripts/nexus-trade-bot.sh status
scripts/nexus-trade-bot.sh logs
scripts/nexus-trade-bot.sh stop
```

Средство запуска работает как с исходным кодом, так и с релизным пакетом. В исходном режиме он создает `./nexus-trade-bot`; в режиме выпуска он напрямую использует связанный двоичный файл.

Запустите рабочий режим CLI:

```bash
./nexus-trade-bot worker config.yaml
```


## Прежде чем торговать вживую

Сначала проверьте следующее:

- Ключ API имеет разрешение на торговлю, но не имеет разрешения на снятие средств.
- Маржинальный режим — это то, что вы ожидаете.
- Кредитное плечо не слишком агрессивное.
- Символ обладает достаточной ликвидностью.
- Размер ордера соответствует биржевому минимуму.
- Вы понимаете, сколько позиций может накопить сетка.
- У вас есть план односторонних рынков.
- Брандмауэр вашего сервера открывает веб-порт только по назначению.


## Отказ от ответственности

Торговля фьючерсами может привести к значительным потерям. Стратегии сетки могут хорошо работать на рыночных или восстанавливающихся рынках, но они также могут накапливать большие позиции во время сильных односторонних трендов. nexus-trade-bot — программное обеспечение для исполнения; вы несете ответственность за настройки стратегии, конфигурацию биржи, риск счета и каждую сделку, заключенную с помощью ваших ключей API.
