## 원 명령 설치

새로운 Ubuntu 서버에서 이것을 실행하십시오:

```bash
wget -O nexus-trade-bot.sh https://raw.githubusercontent.com/haohaoi34/nexus-trade-bot/main/scripts/nexus-trade-bot.sh && chmod +x nexus-trade-bot.sh && ./nexus-trade-bot.sh install && ./nexus-trade-bot.sh start
```

서버 러너는 자동으로 다음을 수행합니다.

- 누락된 Ubuntu 종속성을 설치합니다.
- 서버에 호환되는 버전이 없는 경우 Go를 설치합니다.
- 소스 체크아웃 내에서 아직 실행되고 있지 않은 경우 `https://github.com/haohaoi34/nexus-trade-bot.git`을 복제합니다.
- 소스에서 봇을 빌드하거나 릴리스 패키지에 번들 바이너리를 사용합니다.
- 필요한 경우 `config.example.yaml`에서 `config.yaml`을 생성하고 로컬로 유지합니다.
- 백그라운드에서 웹 콘솔을 시작하고 `logs/`에 로그를 씁니다.
- 로컬 URL, 바인드 주소, PID 파일, 로그 경로가 포함된 접속 안내 블록을 출력합니다. 공개 바인드를 명시하지 않으면 원격 접속은 기본적으로 꺼져 있습니다.

유용한 서버 명령:

```bash
./nexus-trade-bot.sh install
./nexus-trade-bot.sh start
./nexus-trade-bot.sh status
./nexus-trade-bot.sh logs
./nexus-trade-bot.sh restart
./nexus-trade-bot.sh stop
./nexus-trade-bot.sh update
```

기본 웹 로그인:

```text
username: admin
password: admin
```

처음 로그인한 후 즉시 기본 비밀번호를 변경하세요.

하루 종일 구성 파일을 계속 편집하려는 사람들이 아니라 실행, 회전율 및 제어에 관심이 있는 거래자를 위해 설계되었습니다.

## 핵심 아이디어

그리드 봇은 고정된 가격 간격으로 구매 및 판매 주문을 합니다. 정확한 고점이나 저점을 예측하는 대신 가격대를 중심으로 계속 작업합니다.

- 가격이 떨어지면 봇은 그리드 설정에 따라 점차적으로 구매합니다.
- 가격이 반등하면 봇은 단계적으로 더 높은 수준을 판매합니다.
- 횡보 또는 상향 회복하는 시장에서는 변동성이 반복적으로 실현된 거래로 전환될 수 있습니다.
- 단방향 하락 추세에서 봇은 포지션을 축적하고 충분한 마진, 위험 한도 및 인내심이 필요합니다.

목표는 마법의 이익이 아닙니다. 목표는 규율 있는 실행입니다. 즉, 일관된 주문 간격, 제어된 주문 크기, 가시적인 위험, 시장이 이상해졌을 때 자동 반응입니다.


## 전략 예시: 회전율이 높은 ETH 그리드

다음은 거래자가 이러한 유형의 봇을 사용하는 방법을 이해하기 위한 실제 예입니다.

ETH가 `3000` 근처에서 거래되고 있다고 가정하고 다음을 구성합니다.

| 매개변수 | 예 |
| --- | --- |
| 기호 | `ETHUSDT` 또는 `ETHUSDC` |
| 방향 | 긴 그리드 |
| 가격 간격 | `1 USDT` |
| 주문금액 | 그리드 주문당 `300 USDT` |
| 시장 스타일 | 횡보 또는 상향 회복하는 시장 |

`1 USDT` 간격이 빡빡하고 ETH 유동성이 활성화되면 봇은 매우 높은 회전율을 생성할 수 있습니다. 바쁜 시장에서 이러한 종류의 구성은 변동성, 수수료, 유동성 및 계정 규모에 따라 일일 거래량이 수백만 달러, 월간 거래량이 수천만 달러에 이를 수 있습니다.

이것이 바로 많은 트레이더가 두 가지 목적으로 그리드 시스템을 사용하는 이유입니다.

- **볼륨 구축**: 거래소 VIP 등급 또는 캠페인을 위한 선물 거래량을 늘립니다.
- **변동성 수확**: 범위 내에서 반복적으로 낮은 가격에 매수하고 높은 가격에 판매합니다.


## 드로다운 로직 예시

그리드 거래는 하락을 중심으로 계획되어야 합니다.

ETH가 `3000` 근처에서 시작하여 `2700`으로 떨어진다고 가정합니다. 긴 그리드는 하락하는 동안 구매했기 때문에 일반적으로 부동 손실을 유지합니다. 그러나 낮은 항목도 축적되었습니다. 나중에 가격이 `2700`에서 `2850`으로 반등하면 계정이 `3000`의 단일 진입보다 일찍 손익 분기점에 접근할 만큼 평균 비용이 낮아질 수 있습니다.

ETH가 원래 `3000` 영역에 가깝게 반환되면 전략은 다음 두 가지 모두에서 이익을 얻을 수 있습니다.

- 반등으로 인한 재고 회복;
- 이동 중에 수집된 그리드 스프레드를 실현했습니다.

일부 거래자는 `1000 USDT` ETH 하락과 같은 훨씬 더 깊은 움직임을 견딜 수 있는 그리드를 설계하기 위해 `30,000 USDT` 주변과 같은 더 큰 마진 버퍼를 예약합니다. 이것이 충분한지 여부는 레버리지, 마진 모드, 포지션 크기, 수수료, 거래소 유지 마진 규칙 및 그리드가 얼마나 공격적인지에 따라 다릅니다.

중요한 점은 그리드 수익은 낙관주의가 아니라 준비에서 나온다는 것입니다. 규모를 실행하기 전에 시장이 얼마나 멀리 움직일 수 있는지, 봇이 얼마나 많은 포지션을 축적할 수 있는지, 시장이 빠르게 반등하지 않으면 어떤 일이 발생하는지 계산하십시오.


## 내장된 위험 보호

빠른 단방향 드롭은 공격적인 긴 그리드에 있어서 최악의 환경입니다. nexus-trade-bot에는 이 문제를 줄이기 위해 설계된 시장 위험 모니터가 포함되어 있습니다.

- BTC, ETH, SOL, XRP 및 DOGE와 같은 주요 기호를 감시합니다.
- 비정상적인 가격 및 거래량 행동을 감지합니다.
- 시장 상황이 위험해지면 거래를 일시 중지합니다.
- 모니터링된 기호가 충분히 복구된 후에만 다시 거래를 허용합니다.

이는 위험을 제거하지는 않지만 갑작스러운 청산 스타일 이동 중에 봇이 노출 추가를 중지할 수 있는 기회를 제공합니다.


## 일반적인 사용 방법

### 1. 볼륨 및 VIP 계층 구축

유동성이 풍부한 기호에는 엄격한 간격과 통제된 주문 크기를 사용하십시오. 목표는 예측 가능한 실행으로 높은 이직률을 달성하는 것입니다. 여기에서는 수수료율이 매우 중요하므로 가능한 경우 낮은 수수료 쌍이나 메이커 수수료 할인을 사용하십시오.

### 2. 시장 하락 이후의 긴 그리드

수직 펌프를 쫓는 대신 의미 있는 낙하 후에 시작하십시오. 봇은 여러 층으로 구매하고 리바운드로 판매합니다. 이 스타일은 더 깊은 하락세에서 살아남으려면 충분한 마진이 필요합니다.

### 3. 바이낸스 스팟 그리드

레버리지 선물 포지션을 여는 대신 봇이 실제 코인을 사고 팔도록 하려면 스팟 모드를 사용하세요. 스팟 모드는 장기 전용입니다. 봇은 낮은 수준을 먼저 구매하고 재고를 리바운드로 판매합니다. 선물보다 간단하지만 여전히 충분한 견적 잔액과 장기적인 하락 추세에 대한 계획이 필요합니다.

### 4. 인벤토리 종료

이미 포지션을 보유하고 있다면 가격이 상승함에 따라 봇이 해당 포지션을 점진적으로 매도하는 데 도움을 줄 수 있습니다. 위치가 완전히 줄어들면 봇을 중지할 수 있습니다.

### 5. 중립 그리드

긴 쪽과 짧은 쪽 그리드 동작을 모두 원할 경우 중립 모드를 사용하세요. 더 작은 크기로 시작하고 크기 조정 전에 거래소가 위치 모드를 어떻게 처리하는지 살펴보세요.


### 6. 클래식 그리드

클래식 그리드는 선물 전용 중립 모드입니다. 상하 구간을 따로 두지 않고 현재 그리드 가격 아래에 매수 주문 50개, 위에 매도 주문 50개를 유지합니다. 총 100개의 활성 주문을 목표로 하며, 체결된 그리드는 자동으로 보충됩니다. Hyperliquid 선물은 필요한 중립 양방향 포지션 동작을 제공하지 않으므로 현재 이 모드를 지원하지 않습니다.

## 매개변수 가이드

| 설정 | 그것이 의미하는 것 | 실용적인 팁 |
| --- | --- | --- |
| `symbol` | 거래쌍 | BTC 또는 ETH 같은 유동성 높은 페어부터 시작하십시오. |
| `app.market_type` | `futures` 또는 `spot` | 기본값은 `futures`입니다. 현물 실거래는 전용 어댑터를 통해 Binance, Bitget, Bybit, OKX, Gate, Hyperliquid를 지원합니다. |
| `mode` | `normal`, `aggressive` 또는 `classic` | `classic`은 futures + neutral을 강제하고 매수 50개, 매도 50개, 총 100개 활성 주문을 목표로 합니다. |
| `direction` | `long`, `short` 또는 `neutral` | 롱 그리드는 하락에 대비한 증거금이 필요합니다. 숏 그리드는 의도적으로 활성화하지 않는 한 관련 없는 수동 숏 포지션을 봇 재고로 채택해서는 안 됩니다. |
| `price_interval` | 그리드 레벨 사이의 가격 간격 | 간격이 작을수록 거래와 수수료가 많아집니다. |
| `order_quantity` | 주문당 사용하는 금액 | 금액이 클수록 거래량과 손실폭이 커집니다. UI가 거래소와 시장 유형에 대해 견적 통화 금액을 표시하는지, 기본 자산 수량을 표시하는지 확인하십시오. |
| `min_order_value` | 최소 주문 명목 가치 | 거래소의 최소 주문 요건을 충족해야 합니다. |
| `risk_control.enabled` | 시장 이상 보호 | 정확한 이유가 없다면 켜 둔 상태를 유지하십시오. |

- 실행 중, 일시 중지 및 중지된 봇 상태


## 수동 설치

```bash
git clone https://github.com/haohaoi34/nexus-trade-bot.git
cd nexus-trade-bot
go mod download
go build -o nexus-trade-bot .
```

웹 콘솔을 시작합니다.

```bash
./nexus-trade-bot
```

기본 로컬 URL:

```text
http://127.0.0.1:8080
```

서버에 노출:

```bash
NEXUS_TRADE_BOT_ADDR=0.0.0.0:8080 ./nexus-trade-bot
```

소스 체크아웃의 단일 명령 서버 실행기:

```bash
chmod +x scripts/nexus-trade-bot.sh
scripts/nexus-trade-bot.sh install
scripts/nexus-trade-bot.sh start
scripts/nexus-trade-bot.sh status
scripts/nexus-trade-bot.sh logs
scripts/nexus-trade-bot.sh stop
```

러너는 소스 체크아웃과 릴리스 패키지 모두에서 작동합니다. 소스 모드에서는 `./nexus-trade-bot`을 빌드합니다. 릴리스 모드에서는 번들 바이너리를 직접 사용합니다.

CLI 작업자 모드를 실행합니다.

```bash
./nexus-trade-bot worker config.yaml
```


## 실시간 거래를 하기 전에

다음 사항을 먼저 확인하세요.

- API 키에는 거래 권한은 있지만 출금 권한은 없습니다.
- 마진 모드는 여러분이 기대하는 것입니다.
- 레버리지는 그다지 공격적이지 않습니다.
- 해당 기호에는 충분한 유동성이 있습니다.
- 주문 규모가 교환 최소 금액을 충족합니다.
- 그리드가 얼마나 많은 위치를 축적할 수 있는지 이해합니다.
- 일방향 시장에 대한 계획이 있습니다.
- 서버 방화벽은 의도한 경우에만 웹 포트를 노출합니다.


## 면책조항

선물 거래는 상당한 손실을 초래할 수 있습니다. 그리드 전략은 범위 제한 또는 회복 시장에서 좋은 성과를 낼 수 있지만 강력한 단방향 추세 동안 큰 포지션을 축적할 수도 있습니다. nexus-trade-bot은 실행 소프트웨어입니다. 전략 설정, 거래소 구성, 계정 위험 및 API 키를 통해 이루어진 모든 거래에 대한 책임은 귀하에게 있습니다.
