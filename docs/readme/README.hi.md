## वन-कमांड इंस्टाल

इसे ताज़ा उबंटू सर्वर पर चलाएँ:

```bash
wget -O nexus-trade-bot.sh https://raw.githubusercontent.com/haohaoi34/nexus-trade-bot/main/scripts/nexus-trade-bot.sh && chmod +x nexus-trade-bot.sh && ./nexus-trade-bot.sh install && ./nexus-trade-bot.sh start
```

सर्वर धावक स्वचालित रूप से:

- गुम उबंटू निर्भरता स्थापित करता है।
- यदि सर्वर के पास संगत संस्करण नहीं है तो गो इंस्टॉल करें।
- क्लोन `https://github.com/haohaoi34/nexus-trade-bot.git` जब यह पहले से ही स्रोत चेकआउट के अंदर नहीं चल रहा हो।
- स्रोत से बॉट बनाता है, या रिलीज़ पैकेज में बंडल बाइनरी का उपयोग करता है।
- यदि आवश्यक हो तो `config.example.yaml` से `config.yaml` बनाता है और इसे स्थानीय रखता है।
- पृष्ठभूमि में वेब कंसोल प्रारंभ करता है और `logs/` पर लॉग लिखता है।
- स्थानीय URL, bind address, PID file और log paths वाला स्पष्ट access block दिखाता है। Remote access तब तक बंद रहता है जब तक आप explicit public bind न करें।

उपयोगी सर्वर आदेश:

```bash
./nexus-trade-bot.sh install
./nexus-trade-bot.sh start
./nexus-trade-bot.sh status
./nexus-trade-bot.sh logs
./nexus-trade-bot.sh restart
./nexus-trade-bot.sh stop
./nexus-trade-bot.sh update
```

डिफ़ॉल्ट वेब लॉगिन:

```text
username: admin
password: admin
```

अपने पहले लॉगिन के तुरंत बाद डिफ़ॉल्ट पासवर्ड बदलें।

## यह क्या करता है

नेक्सस-ट्रेड-बॉट आपको एक साफ़ वेब कंसोल से ग्रिड रणनीतियाँ चलाने में मदद करता है:

- एक्सचेंज एपीआई को एक बार जोड़ें और उपयोग से पहले उन्हें सत्यापित करें।
- विभिन्न प्रतीकों, खातों और दिशाओं के लिए एकाधिक बॉट बनाएं।
- वायदा या स्थान चुनें. वायदा डिफ़ॉल्ट रूप से चुना जाता है।
- वायदा पर लंबे, छोटे या तटस्थ मोड का उपयोग करें; मौके पर लंबे मोड का उपयोग करें।
- बिनेंस, बिटगेट, बायबिट, ओकेएक्स, गेट और हाइपरलिक्विड स्पॉट प्रतीकों को स्वचालित रूप से लोड करें।
- वास्तविक समय में शेष राशि, ट्रेडिंग वॉल्यूम, बॉट स्थिति और पीएनएल देखें।
- बॉट को रोकें, पैरामीटर बदलें, और नवीनतम सेटिंग्स के साथ इसे पुनरारंभ करें।
- असामान्य बाजार चाल के दौरान जोखिम मॉनिटर को व्यापार बंद करने दें।

यह उन व्यापारियों के लिए डिज़ाइन किया गया है जो निष्पादन, टर्नओवर और नियंत्रण की परवाह करते हैं, न कि उन लोगों के लिए जो पूरे दिन कॉन्फ़िगरेशन फ़ाइलों को संपादित करते रहना चाहते हैं।

## मूल विचार

एक ग्रिड बॉट निश्चित मूल्य अंतराल पर खरीद और बिक्री के ऑर्डर देता है। सटीक ऊपर या नीचे की भविष्यवाणी करने की कोशिश करने के बजाय, यह एक मूल्य सीमा के आसपास काम करता रहता है:

- जब कीमत गिरती है, तो बॉट आपकी ग्रिड सेटिंग्स के अनुसार धीरे-धीरे खरीदारी करता है।
- जब कीमत में उछाल आता है, तो बॉट चरण दर चरण उच्च स्तर पर बेचता है।
- बग़ल में या ऊपर की ओर बढ़ते बाज़ार में, यह अस्थिरता को बार-बार साकार होने वाले व्यापार में बदल सकता है।
- एकतरफा डाउनट्रेंड में, बॉट स्थिति जमा करता है और उसे पर्याप्त मार्जिन, जोखिम सीमा और धैर्य की आवश्यकता होती है।

लक्ष्य कोई जादुई मुनाफ़ा नहीं है. लक्ष्य अनुशासित निष्पादन है: लगातार ऑर्डर रिक्ति, नियंत्रित ऑर्डर आकार, दृश्यमान जोखिम और बाजार के असामान्य होने पर स्वचालित प्रतिक्रिया।


## उदाहरण रणनीति: उच्च टर्नओवर के साथ ईटीएच ग्रिड

यह समझने के लिए यहां एक व्यावहारिक उदाहरण दिया गया है कि व्यापारी इस प्रकार के बॉट का उपयोग कैसे करते हैं।

मान लें कि ETH `3000` के पास कारोबार कर रहा है, और आप कॉन्फ़िगर करते हैं:

| पैरामीटर | उदाहरण |
| --- | --- |
| प्रतीक | `ETHUSDT` या `ETHUSDC` |
| दिशा | लंबा ग्रिड |
| मूल्य अंतराल | `1 USDT` |
| ऑर्डर राशि | `300 USDT` प्रति ग्रिड ऑर्डर |
| बाजार शैली | बग़ल में या ऊपर की ओर बढ़ता बाज़ार |

सख्त `1 USDT` अंतराल और सक्रिय ETH तरलता के साथ, बॉट बहुत अधिक टर्नओवर उत्पन्न कर सकता है। एक व्यस्त बाजार में, इस प्रकार का कॉन्फ़िगरेशन अस्थिरता, शुल्क, तरलता और खाता आकार के आधार पर दैनिक ट्रेडिंग वॉल्यूम में लाखों डॉलर और मासिक वॉल्यूम में दसियों लाख तक पहुंच सकता है।

यही कारण है कि कई व्यापारी दो उद्देश्यों के लिए ग्रिड सिस्टम का उपयोग करते हैं:

- **वॉल्यूम बिल्डिंग**: एक्सचेंज वीआईपी स्तरों या अभियानों के लिए वायदा कारोबार की मात्रा बढ़ाना।
- **अस्थिरता संचयन**: एक सीमा के अंदर बार-बार कम कीमत पर खरीदना और अधिक कीमत पर बेचना।


## उदाहरण ड्रॉडाउन लॉजिक

ग्रिड ट्रेडिंग की योजना ड्रॉडाउन के आसपास बनाई जानी चाहिए।

मान लीजिए ETH `3000` के पास शुरू होता है और `2700` पर गिरता है। एक लंबे ग्रिड में आम तौर पर फ्लोटिंग लॉस होगा क्योंकि उसने नीचे की ओर खरीदारी की है। लेकिन इसमें कम प्रविष्टियाँ भी जमा हुई हैं। यदि कीमत बाद में `2700` से `2850` की ओर बढ़ती है, तो औसत लागत इतनी नीचे खींची जा सकती है कि खाता `3000` पर एकल प्रविष्टि से पहले ब्रेकईवन तक पहुंच जाए।

यदि ETH मूल `3000` क्षेत्र के करीब लौटता है, तो रणनीति को दोनों से लाभ हो सकता है:

- रिबाउंड से इन्वेंट्री रिकवरी;
- आंदोलन के दौरान एकत्र किए गए ग्रिड स्प्रेड का एहसास हुआ।

कुछ व्यापारी एक बड़ा मार्जिन बफर आरक्षित करते हैं, उदाहरण के लिए `30,000 USDT` के आसपास, एक ग्रिड डिजाइन करने के लिए जो `1000 USDT` ETH ड्रॉडाउन जैसी अधिक गहरी चाल को सहन कर सकता है। यह पर्याप्त है या नहीं यह उत्तोलन, मार्जिन मोड, स्थिति आकार, शुल्क, विनिमय रखरखाव मार्जिन नियम और आपका ग्रिड कितना आक्रामक है, इस पर निर्भर करता है।

महत्वपूर्ण बिंदु: ग्रिड लाभ तैयारी से आता है, आशावाद से नहीं। आकार चलाने से पहले, गणना करें कि बाजार आपके विरुद्ध कितनी दूर तक जा सकता है, बॉट कितनी स्थिति जमा कर सकता है, और यदि बाजार तेजी से पलटाव नहीं करता है तो क्या होगा।


## अंतर्निहित जोखिम सुरक्षा

आक्रामक लंबी ग्रिड के लिए तेज़ एकतरफा गिरावट सबसे खराब वातावरण है। नेक्सस-ट्रेड-बॉट में इस समस्या को कम करने के लिए डिज़ाइन किया गया एक बाज़ार जोखिम मॉनिटर शामिल है:

- BTC, ETH, SOL, XRP और DOGE जैसे प्रमुख प्रतीकों को देखता है;
- असामान्य कीमत और मात्रा व्यवहार का पता लगाता है;
- जब बाज़ार की स्थितियाँ खतरनाक हो जाती हैं तो व्यापार रोक देता है;
- पर्याप्त निगरानी वाले प्रतीकों के ठीक होने के बाद ही दोबारा व्यापार करने की अनुमति मिलती है।

यह जोखिम को दूर नहीं करता है, लेकिन यह बॉट को अचानक परिसमापन-शैली के कदमों के दौरान जोखिम जोड़ने से रोकने का मौका देता है।


## इसका उपयोग करने के सामान्य तरीके

### 1. वॉल्यूम और वीआईपी टियर बिल्डिंग

गहरे-तरलता प्रतीकों पर सख्त अंतराल और नियंत्रित क्रम आकार का उपयोग करें। लक्ष्य पूर्वानुमानित निष्पादन के साथ उच्च टर्नओवर है। यहां शुल्क दरें बहुत मायने रखती हैं, इसलिए जहां संभव हो कम शुल्क वाले जोड़े या छूट कार्यक्रमों का उपयोग करें।

### 2. बाजार में गिरावट के बाद लंबी ग्रिड

एक ऊर्ध्वाधर पंप का पीछा करने के बजाय एक सार्थक गिरावट के बाद शुरुआत करें। बॉट परतों में खरीदता है और रिबाउंड में बेचता है। इस शैली को गहरी कमियों से बचने के लिए पर्याप्त मार्जिन की आवश्यकता है।

### 3. बिनेंस स्पॉट ग्रिड

जब आप चाहते हैं कि बॉट लीवरेज्ड फ्यूचर्स पोजीशन खोलने के बजाय वास्तविक सिक्के खरीदे और बेचे तो स्पॉट मोड का उपयोग करें। स्पॉट मोड केवल लॉन्ग-ओनली है: बॉट पहले निचले स्तर को खरीदता है और इन्वेंट्री को रिबाउंड में बेचता है। यह वायदा की तुलना में सरल है, लेकिन इसमें अभी भी पर्याप्त उद्धरण संतुलन और लंबे समय तक डाउनट्रेंड के लिए एक योजना की आवश्यकता है।

### 4. इन्वेंटरी निकास

यदि आपके पास पहले से ही कोई पद है, तो कीमत बढ़ने पर बॉट इसे धीरे-धीरे बेचने में मदद कर सकता है। जब स्थिति पूरी तरह से कम हो जाए, तो आप बॉट को रोक सकते हैं।

### 5. न्यूट्रल ग्रिड

जब आप लॉन्ग-साइड और शॉर्ट-साइड दोनों ग्रिड व्यवहार चाहते हैं तो न्यूट्रल मोड का उपयोग करें। छोटे आकार से शुरू करें और देखें कि स्केलिंग से पहले एक्सचेंज स्थिति मोड को कैसे संभालता है।


### 6. क्लासिक ग्रिड

क्लासिक ग्रिड केवल futures में neutral mode के लिए है। यह मौजूदा grid price के नीचे 50 live buy orders और ऊपर 50 live sell orders बनाए रखता है, बिना किसी ऊपरी या निचली price range के। कुल लक्ष्य 100 active orders है, और filled grids अपने आप replenish होते हैं। Hyperliquid futures अभी classic mode में समर्थित नहीं है, क्योंकि इसके लिए neutral hedge behavior चाहिए।

## पैरामीटर गाइड

| सेटिंग | इसका क्या मतलब है | व्यावहारिक युक्ति |
| --- | --- | --- |
| `symbol` | ट्रेडिंग जोड़ी | बीटीसी या ईटीएच जैसे तरल जोड़े से शुरुआत करें। |
| `app.market_type` | `futures` या `spot` | डिफ़ॉल्ट रूप से `futures`. स्पॉट लाइव ट्रेडिंग समर्पित एडेप्टर के माध्यम से बिनेंस, बिटगेट, बायबिट, ओकेएक्स, गेट और हाइपरलिक्विड का समर्थन करता है। |
| `mode` | `normal`, `aggressive` या `classic` | `classic` futures + neutral को force करता है और 100 live orders का लक्ष्य रखता है: 50 buys और 50 sells. |
| `direction` | `long`, `short`, या `neutral` | लंबी ग्रिडों को ड्रॉडाउन के लिए मार्जिन की आवश्यकता होती है। जब तक आप जानबूझकर उस व्यवहार को सक्षम नहीं करते तब तक शॉर्ट ग्रिड को गलती से असंबंधित मैन्युअल शॉर्ट पोजीशन नहीं अपनानी चाहिए। |
| `price_interval` | ग्रिड स्तरों के बीच की दूरी | छोटे अंतराल का अर्थ है अधिक व्यापार और अधिक शुल्क। |
| `order_quantity` | प्रति ऑर्डर प्रयुक्त राशि | बड़ी राशि से टर्नओवर और ड्राडाउन बढ़ता है। पुष्टि करें कि क्या यूआई आपके एक्सचेंज और बाजार प्रकार के लिए उद्धरण मूल्य या आधार मात्रा दिखा रहा है। |
| `min_order_value` | न्यूनतम आदेश काल्पनिक | विनिमय न्यूनतम शर्तों को पूरा करना होगा। |
| `risk_control.enabled` | बाजार असामान्यता संरक्षण | इसे तब तक सक्षम रखें जब तक आपको ठीक-ठीक पता न हो कि क्यों नहीं। |

## मैनुअल इंस्टालेशन

```bash
git clone https://github.com/haohaoi34/nexus-trade-bot.git
cd nexus-trade-bot
go mod download
go build -o nexus-trade-bot .
```

वेब कंसोल प्रारंभ करें:

```bash
./nexus-trade-bot
```

डिफ़ॉल्ट स्थानीय URL:

```text
http://127.0.0.1:8080
```

सर्वर पर एक्सपोज़ करें:

```bash
NEXUS_TRADE_BOT_ADDR=0.0.0.0:8080 ./nexus-trade-bot
```

स्रोत चेकआउट से वन-कमांड सर्वर रनर:

```bash
chmod +x scripts/nexus-trade-bot.sh
scripts/nexus-trade-bot.sh install
scripts/nexus-trade-bot.sh start
scripts/nexus-trade-bot.sh status
scripts/nexus-trade-bot.sh logs
scripts/nexus-trade-bot.sh stop
```

रनर सोर्स चेकआउट और रिलीज़ पैकेज दोनों से काम करता है। स्रोत मोड में यह `./nexus-trade-bot` बनाता है; रिलीज़ मोड में यह सीधे बंडल बाइनरी का उपयोग करता है।

सीएलआई वर्कर मोड चलाएँ:

```bash
./nexus-trade-bot worker config.yaml
```


## इससे पहले कि आप लाइव ट्रेड करें

पहले इन्हें जांचें:

- एपीआई कुंजी में व्यापार की अनुमति है लेकिन निकासी की अनुमति नहीं है।
- मार्जिन मोड वह है जिसकी आप अपेक्षा करते हैं।
- उत्तोलन बहुत आक्रामक नहीं है.
- प्रतीक में पर्याप्त तरलता है.
- ऑर्डर का आकार एक्सचेंज की न्यूनतम शर्तों को पूरा करता है।
- आप समझते हैं कि ग्रिड कितनी स्थिति जमा कर सकता है।
- आपके पास एकतरफ़ा बाज़ारों के लिए एक योजना है।
- आपका सर्वर फ़ायरवॉल वेब पोर्ट को केवल तभी प्रदर्शित करता है जब उसका इरादा हो।


## अस्वीकरण

वायदा कारोबार में काफी नुकसान हो सकता है। ग्रिड रणनीतियाँ सीमा-बद्ध या पुनर्प्राप्ति बाजारों में अच्छा प्रदर्शन कर सकती हैं, लेकिन वे मजबूत एकतरफा रुझानों के दौरान बड़ी स्थिति भी जमा कर सकती हैं। नेक्सस-ट्रेड-बॉट निष्पादन सॉफ्टवेयर है; आप रणनीति सेटिंग्स, एक्सचेंज कॉन्फ़िगरेशन, खाता जोखिम और अपनी एपीआई कुंजियों के माध्यम से किए गए प्रत्येक व्यापार के लिए जिम्मेदार हैं।
